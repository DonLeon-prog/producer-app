# Контекст проекта — AI-Продюсер Mini App

## Что мы строим
Telegram Mini App — AI-продюсер для двух типов пользователей:
- **Блогеры** — снимают короткие видео, хотят вырасти в соцсетях
- **Музыканты / певцы** — записывают треки, хотят продвинуться на стримингах и набрать аудиторию

Пользователь при онбординге выбирает кто он. Все AI-функции, контент-планы и советы разные для каждого типа.

## Стек
- Frontend: React 18 + TypeScript + Vite + TailwindCSS + Zustand
- Backend: Python FastAPI + PostgreSQL + Redis
- AI: Google Gemini 2.0 Flash API (бесплатный)
- Auth: Telegram initData → JWT
- Оплата: Telegram Stars
- Деплой: Railway (бэк) + Vercel (фронт)

## Структура репозитория
- /frontend — React приложение (Telegram Mini App)
- /backend — FastAPI сервер
- /docs — документация
- SPRINTS.md — подробный план по спринтам
- CLAUDE.md — этот файл, контекст для Claude Code

## Архитектура пользователей
В таблице users есть поле user_type: "blogger" | "musician"
Поле profile (JSON) содержит разные данные в зависимости от user_type:

Блогер:
  name, niches[], platforms[], inspiration, goals[]

Музыкант:
  name, genre[], platforms[], influences, release_status, goals[]

## AI промпты
- backend/app/services/prompts.py — промпты для блогеров
- backend/app/services/prompts_musician.py — промпты для музыкантов
- backend/app/services/gemini.py — два системных промпта: SYSTEM_BLOGGER и SYSTEM_MUSICIAN

## Онбординг
Первый экран: выбор типа (UserTypeSelect)
  → Блогер: 5 шагов (имя, ниша, платформы, вдохновение, цель)
  → Музыкант: 6 шагов (имя, жанр, платформы, вдохновение, статус, цель)

## AI-функции по типам

Блогер:
  - Контент-план на 7 дней (короткие видео)
  - Идеи для видео
  - Подписи к постам + хэштеги
  - Анализ конкурента-блогера

Музыкант:
  - Контент-план на 7 дней (контент вокруг музыки)
  - Идеи для контента
  - Подписи к постам о треках
  - План продвижения релиза на 3 недели
  - Питч для лейбла / продюсера
  - Питч куратору плейлиста
  - Анализ похожего артиста

## Killer-фича: Growth Engine

### Челленджи (как Duolingo, но для контента)
После онбординга пользователь запускает 7-дневный челлендж.
Каждый день — одно конкретное задание. Выполнил → галочка → AI-фидбек → следующее.
Задания разные для блогеров и музыкантов.
Компоненты: DailyChallenge (карточка на дашборде), модалка с отметкой и фидбеком.

### Growth Map (трекер роста с ручным вводом)
Пользователь сам вводит метрики каждый день: просмотры, лайки, подписчики.
Никаких интеграций с Instagram/TikTok API — они заблокированы и слишком сложны.
Приложение показывает bar-chart за 7 дней и AI-совет на основе реальных данных.
Компоненты: отдельная вкладка "Рост" в таббаре, форма ввода, чарт, AI-инсайт.

### Таблицы в БД
- challenges: id, user_id, user_type, started_at, status, current_day, completed_days (JSON)
- growth_metrics: id, user_id, date, platform, views, likes, followers_gained, streams, saves, notes
