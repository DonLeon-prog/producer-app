---

## Как пользоваться этим планом

1. Открываешь Claude Code в терминале (`claude` в папке проекта)
2. Берёшь задачу из спринта
3. Вставляешь её текст как есть — Claude Code сам разберётся
4. Проверяешь что получилось
5. Переходишь к следующей задаче

## Спринт 1 — Создаём проект и запускаем локально
**Срок: неделя 1–2 | ~30 часов | Цель: проект работает у тебя на компьютере**

Это фундамент. После этого спринта у тебя будет живой проект — бэкенд отвечает, фронт открывается в браузере, база данных работает. Ничего красивого, просто каркас.

### Что нужно установить один раз (сделай руками, не через Claude Code)

- **Node.js** — скачать с nodejs.org, версия 20+
- **Python 3.11+** — скачать с python.org
- **Docker Desktop** — скачать с docker.com (нужен для базы данных локально)
- **GitHub CLI** — скачать с cli.github.com
- **Claude Code** — после установки Node: `npm install -g @anthropic-ai/claude-code`

Проверь что всё стоит:
```bash
node --version    # должно быть v20+
python3 --version # должно быть 3.11+
docker --version  # любая версия
gh --version      # любая версия
claude --version  # любая версия
```

### Задача 2.1 — Авторизация через Telegram

> Когда пользователь открывает Mini App, Telegram передаёт данные о нём (кто он, его id). Нам нужно это проверить и выдать JWT токен — это как пропуск, который фронт будет прикладывать к каждому запросу.

**Вставь в Claude Code:**
```
Реализуй авторизацию через Telegram initData.

В backend/app/core/auth.py:
- Функция validate_telegram_init_data(init_data: str, bot_token: str) -> dict
  Проверяет подпись через HMAC-SHA256 (стандартный алгоритм Telegram)
  Возвращает распарсенные данные пользователя или выбрасывает HTTPException 401
- Функция create_jwt_token(user_id: str) -> str
  Создаёт JWT токен с user_id и сроком жизни 30 дней
- Функция get_current_user(token) — FastAPI dependency
  Проверяет JWT из заголовка Authorization: Bearer <token>
  Возвращает user_id или 401

В backend/app/api/auth.py:
- POST /api/auth/telegram
  Принимает {"init_data": "..."}
  Проверяет initData, находит или создаёт пользователя в БД
  Возвращает {"access_token": "...", "user": {...}}

В backend/app/schemas/user.py создай схемы:
- TelegramAuthRequest
- UserResponse  
- AuthResponse
```
